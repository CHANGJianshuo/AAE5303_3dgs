## 二、LiDAR 数据 (`lidar/`)

| 属性 | 值 |
|------|-----|
| 数量 | **7,498 帧** |
| 传感器 | **Livox**（大疆旗下激光雷达） |
| 消息类型 | `livox_ros_driver/CustomMsg` |
| 单帧大小 | 固定 **1,336 KB** |
| 总大小 | **9.55 GB** |
| 帧率 | ~10 fps（和相机同步） |
| 格式 | Livox CustomMsg 序列化的 raw bytes |

### 点云估算

每帧 1,336 KB，按 Livox CustomMsg 点格式（每点约 19 bytes：xyz float32 + reflectivity uint8 + tag uint8 + line uint8）：
- 每帧约 **~72,000 个点**
- 7,498 帧合计约 **5.4 亿个点**（540 million）

### 与 baseline 的对比

baseline 的 `merge_lidar_to_ply.py` 对 1,380 个 LiDAR txt 文件处理：
- 原始 34M 点 -> 每10个取1个 -> 3.4M 点 -> cloud_merged.ply (49 MB)
- 转 COLMAP 时再每10个取1个 -> 34 万点写入 points3D.bin

本次提取的 raw bytes 需要先解析 Livox CustomMsg 格式再转为 xyz 坐标。

## 三、姿态数据 (`attitudes.json`)

| 属性 | 值 |
|------|-----|
| 数量 | **74,992 条** |
| 频率 | **100 Hz** |
| 格式 | 四元数 `(qw, qx, qy, qz)` |
| 来源 | DJI 飞控 IMU 融合输出 |

### 数据示例

```json
{
    "timestamp": 1658132896117167514,
    "qw": 0.4494,
    "qx": -0.0027,
    "qy": 0.0027,
    "qz": 0.8933
}
```

### 解读

- 这是**无人机机体姿态**（body frame），不是相机姿态
- `qw` 和 `qz` 为主分量，说明机体主要绕 Z 轴旋转（偏航）
- `qx` 和 `qy` 接近 0，表示俯仰和横滚很小（飞行平稳）
- 要得到相机姿态，还需要叠加**云台角度**的旋转

| 属性 | 值 |
|------|-----|
| 数量 | **37,496 条** |

| X 范围 | 3,826,721 ~ 3,827,520 m（跨度 **799 m**） |

### 飞行轨迹

```

   0s      0.0        0.0      1051     起飞点

 300s      1.7      260.7      1131     转弯
 360s    133.4      213.1      1131     继续扫描
 420s    482.1      399.5      1131     到达边缘
```

### 飞行模式特征

- **往返式航线扫描（Lawnmower pattern）**
- 覆盖面积：**800m x 590m ≈ 0.47 km²**
- 巡航高度：~80m AGL（恒定）
- 起终点几乎重合（距离 0.2m），说明是自动返航
- 总飞行时间：12.5 分钟

## 六、云台角度 (`gimbal_angles.json`)

| 属性 | 值 |
|------|-----|
| 数量 | **37,496 条** |
| 频率 | **50 Hz** |
| Roll 范围 | **-90.00 ~ 0.10** |
| Pitch 范围 | **0.00 ~ 0.00**（始终为 0） |
| Yaw 范围 | **-179.80 ~ 179.90** |

### 数据示例

```json
{
    "timestamp": 1658132896117167514,
    "roll": 0.00,
    "pitch": 0.00,
    "yaw": -32.60
}
```

### 关键发现

| 状态 | 占比 | 说明 |
|------|------|------|
| Roll = -90 (相机正朝下) | **92.7%** | 典型航测/测绘模式，正射拍摄 |
| Roll = 0 (相机朝前) | **7.3%** | 起飞/降落阶段 |

### 云台时间线

```
t=0s     roll=0     yaw=-33    <- 起飞，相机朝前
t=60s    roll=-90   yaw=-21    <- 到达巡航高度，云台转为俯视
t=120s   roll=-90   yaw=67     <- 航线飞行中
t=180s   roll=-90   yaw=67
t=240s   roll=-90   yaw=-113   <- 转弯，yaw 变化
t=300s   roll=-90   yaw=-113
...
t=720s   roll=-90   yaw=-34    <- 返回起飞点
```

### 含义

- **绝大多数照片是正射（俯视）视角**，非常适合做地面/建筑的 3D 重建
- Pitch 始终为 0，说明云台只做了 roll（俯仰到正下方）和 yaw（偏航跟随）调整
- Yaw 在 -180~180 全范围变化，说明无人机在转弯时云台会跟随调整朝向
- 起飞阶段（前 ~30秒）的照片是前视的，可能需要在预处理时**剔除**

## 七、数据时间同步关系

```
时间轴 (749.8秒)
|-- 图像:     ████████████████████████  7,499帧  @ 10Hz
|-- LiDAR:    ████████████████████████  7,498帧  @ 10Hz   (和图像1:1对应)
|-- 姿态:     ████████████████████████  74,992条  @ 100Hz  (每张图对应~10条)
|-- GPS:      ████████████████████████  37,496条  @ 50Hz   (每张图对应~5条)
|-- 本地位置: ████████████████████████  37,496条  @ 50Hz
|-- 云台:     ████████████████████████  37,496条  @ 50Hz   (每张图对应~5条)
```

- 图像和 LiDAR 是 **1:1 同步**的（7,499 vs 7,498，差1帧可忽略）
- 位姿数据频率更高，需要**按时间戳插值**匹配到每张图片
- 所有数据共享同一 ROS 时间基准

## 八、从提取数据到 3DGS 的处理步骤

1. **按时间戳匹配**：为每张图片找到最近的 GPS 位置 + 姿态 + 云台角度
2. **计算相机外参**：无人机位置(GPS) + 机体姿态(attitude quaternion) + 云台角度(gimbal) -> 相机在世界坐标系中的旋转矩阵 R 和平移向量 t
3. **合并 LiDAR**：解析 Livox CustomMsg 二进制格式 -> 提取 xyz 坐标 -> 用对应位姿变换到世界坐标系 -> 合并成一个大点云
4. **转 COLMAP 格式**：生成 cameras.bin + images.bin + points3D.bin
5. **（可选）每5帧采样**：7,499 -> ~1,500 张，和 baseline 一致

### 图片采样对训练时间的影响

| 方案 | 图片数 | A100 30k步预计时间 |
|------|--------|-------------------|
| 每5帧采样（和 baseline 一致） | ~1,500 | **30-60 分钟** |
| 每3帧采样 | ~2,500 | **50-90 分钟** |
| 全部使用 | 7,499 | **2.5-5 小时** |
